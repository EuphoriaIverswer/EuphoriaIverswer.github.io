Feasibility analysis of local fine-tuning of ChatGLM4 large model 6B version

Introduction

Hardware Resources

  • GPU Nvidia RTX3050 laptop 4GB
  • CPU Intel Core i5 11260H
  • RAM DDR4 16GB
  • SSD 1.5T

The author’s graduation project focuses on addressing the following challenges in knowledge graph construction:

  • Definition of entities and relationships
  • Extraction of entities and relationships
  • Construction of knowledge subgraphs

When utilizing large language models (LLMs) to assist knowledge graph generation, the model should accomplish:

  • Automatic discovery of entities and relationships
  • Automated entity extraction
  • Hierarchical knowledge organization
  • Automated construction of knowledge subgraphs

However, for constructing knowledge graphs in the Computer Science and Technology domain, it requires enhancing the model’s performance in this vertical field.

Initially, two fundamental approaches were considered:

  • Fine-tuning & PEFT (Parameter-Efficient Fine-Tuning)
  • RAG (Retrieval-Augmented Generation)

Given limited temporal and computational resources, the feasibility of fine-tuning appears questionable. This report validates this perspective through two analytical dimensions.

Hardware Resource Analysis for Fine-Tuning

Although ChatGLM4’s paper doesn’t specify optimizer choices, based on ChatGLM3 and other open-source LLMs, we assume the probable use of AdamW optimizer variants. Our analysis takes Adam as an example.

Experiment Result

Analysis from ChatGLM4 Architecture Perspective

Model Architecture of ChatGLM4

Based on the technical report “ChatGLM: From GLM-130B to GLM-4 All Tools”, the core improvements since GLM3 include:

  • ​**Architecture upgrade to RetNet-based efficient structure** supporting million-level context windows with reduced inference latency
  • Capability expansion to multimodal agent systems enabling cross-modal interactions (text/image/audio) and dynamic external tool invocation (APIs/databases)
  • Training optimization through curriculum learning strategies for phased complex task mastery, plus model compression/quantization achieving 50% faster inference speed and edge deployment

Feasibility and Challenges of PEFT on RetNet Architecture

Adaptation Challenges of Retention Mechanism

Problem: Retention models sequence dependencies through decay matrices and recurrent states. Traditional PEFT methods (e.g., LoRA) designed for Self-Attention weight matrices may not directly adapt to Retention’s parameter structure.
Case Example: The parallel (training) vs. recurrent (inference) mode switching in Retention requires consistent PEFT parameter behavior across modes. Failure causes post-fine-tuning instability.

Compatibility of Recurrent States with PEFT

Problem: RetNet’s inference relies on recursively transmitted hidden states. Partial parameter tuning via PEFT (e.g., adapter injection) may disrupt state continuity.
Observed Phenomenon: In long-context dialogue tasks, PEFT-tuned RetNet models exhibit context forgetting or logical discontinuity.

Conclusion

Given the project’s constraints in hardware resources and timeline, the fine-tuning approach is not recommended as a viable technical pathway.
